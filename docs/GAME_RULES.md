# GAME RULES(実装済みルール)

仕様書 §6・§9〜§16 を実装した正式ルール(v2: サバイバル制・特殊ブロック・TYPE BURST・CPU対戦を追加。D-012〜D-015)。
数値は全て `packages/game-core/src/config.ts` で変更できる。

## 盤面

- 6列 × 表示10行。row 0 が最下段
- 危険ライン: 上から2行目(row 8)
- 初期配置: 下から6行(36ブロック)
- 初期盤面に同属性4連結なし・同一問題の重複なし(Seed から決定論的に生成)

## 属性

| 属性 | 形状 |
|------|------|
| 炎 | 三角 |
| 水 | 円 |
| 風 | ひし形 |
| 光 | 星 |

色だけで識別させない(色覚対応)。妨害ブロックは無属性・鍵アイコン。

## 入力とターゲット選択

1. 最初のキーで、そのキーから始まる全ブロックが候補として発光
2. 入力を続けると候補が絞られる
3. 候補が1個になるとロック(白枠+グロー)
4. 入力完了で爆発
5. 候補ゼロになるキーはミス。選択バッファのみリセット(ロック後のミスは進捗を消さない)

複数のローマ字表記(shi/si、chi/ti、tsu/tu、fu/hu、ji/zi、sha/sya、cha/tya、ja/jya/zya、nn/n'/n、xtu/ltu、分解入力 sixya 等)を受理する。「ん」の単独 n は次のかなが母音・な行・や行以外の場合のみ有効。

## 消去

- 入力完了ブロックと上下左右で連結する同属性グループが**3個以上**なら全体消去、1〜2個なら入力ブロックのみ消去(斜めは連結に含めない)
- 消去に隣接する妨害ブロックも破壊される
- 消去後に重力落下

## 自動連鎖

- 落下完了後、同属性**4個以上**の連結グループを全て同時消去
- 消去→落下→判定を安定するまで繰り返す
- 連鎖中は新規入力を受け付けず、行上昇も停止する

### CHAIN 表示

- 直接3個以上消去 = `1 CHAIN`
- 直接1個のみ = 表示なし
- 以降の自動消去で +1(単独消去後に初めて起きた自動消去は `1 CHAIN`)

### 演出時間

ヒットストップ 70ms → 消去 150ms → 落下 120ms(1段階 270ms)

## 特殊ブロック(D-013、バランス調整 D-019)

| 種類 | 出現率 | 効果 |
|------|--------|------|
| ボム(黒球+火花) | 新規行の7% | 入力完了で周囲3×3を爆破(1 CHAIN 起点) |
| プリズム(虹) | 新規行の3.5% | 入力完了で盤面最多属性を全消し(1 CHAIN 起点) |

どちらも爆破後の落下から自動連鎖につながる。1行に特殊ブロックは最大1個、かつ**盤面全体で種類ごとに同時に最大1個**(D-019)。読み仮名は通常ブロックと同じ Tier 分布(short/standard/long)から選ばれ、短文固定ではない。

## TYPE BURST(必殺技、D-013)

- ブロック消去(+3)・連鎖(深度×7)・PERFECT(+6)でゲージが溜まる(最大100)
- 満タン時に **Enter** で発動: 下3行を全て吹き飛ばす(+1000点)
- バースト起因の消去・連鎖ではゲージは溜まらない

## 新規行の落下(D-016)

- 新しい行は**盤面の上から降ってきて**、各列の山の一番上に積まれる。既存ブロックは動かない
- サバイバル: 5.5秒間隔から始まり、経過1秒ごとに25msずつ短縮(最短2.4秒)
- CPU対戦: 6.0秒間隔から始まり最短3.2秒
- 1.5秒前に予告(下端の発光+警告音)
- 新規行だけで4連結を作らない(既存盤面との接続で連鎖候補になるのは許可)
- 積み先が最上段を超えたら OVERFLOW(サバイバル=終了、対戦=敗北)

## 選択キャンセル(D-017)

**Esc** または **Backspace** で入力中・ロック中の選択を解除できる。ミスに数えない。

## ボーナスシステム(D-018)

- **ALL CLEAR**: 盤面全消しで +5000点・ゲージ+50。次の行がすぐ降ってくる
- **LEVEL**(サバイバル): 30秒生存ごとに +レベル×300点
- **ランク**(サバイバル結果): S+ 50000 / S 32000 / A 20000 / B 12000 / C 6000 / D

## モード

### サバイバル(1人用)

時間制限なし。加速する行上昇に耐え、あふれるまでスコアを稼ぐ(D-012)。

### CPU対戦(D-014)

- 難易度: よわい(120KPM)/ ふつう(220KPM)/ つよい(360KPM)。ミス率・思考時間も変わる
- CPU は実際にキーを1打ずつ入力する(結果の直接書き換えなし)
- 連鎖の攻撃力が妨害ブロックに変換され相手盤面へ降る。自分への着弾予告中に攻撃すると相殺
- 時間制限なし。相手の盤面をあふれさせたら勝ち

## スコア

```
score = 正確キー数 × 10
      + 色付きブロック消去 × 100
      + 妨害ブロック破壊 × 120
      + 連鎖深度² × 500
      + PERFECT文章 × 150
```

PERFECT = 1文章をミスなしで打ち切ること。

## 攻撃力(CPU対戦で稼働中)

```
basePower = 色付き消去数 + 妨害破壊数 × 0.5
chainBonus = [0, 0, 2, 5, 9, 14, 20, 27, 35, 44][chain]
simultaneousBonus = max(0, 最大グループ − 3)
perfectBonus = PerfectStreak ≥ 5 ? 2 : 0
送信妨害数 = floor(攻撃力 / 4)  ※1処理上限 18
```

## 指標

- `KPM = 正確キー数 / 実プレイ秒数 × 60`
- `正確率 = 正確キー数 / (正確キー数 + ミスキー数)`
