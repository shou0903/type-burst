# CONTENT GUIDELINES

問題文の作成・追加の指針(仕様書 §7・§26 準拠)。

## 原則

標準問題は**自然な日本語の語句・短文・文章**のみ。全て自作(source: "original")。

### 入れてよい

日常 / 学校 / 仕事 / 移動 / 季節 / 食事 / 会話 / 行動 / 前向きな表現

### 入れない

- 英字だけの文字列、URL、メールアドレス、ファイル名、コード断片
- 歌詞・小説・漫画・映画の台詞、実在人物の発言
- 差別・侮辱・性的表現、政治・宗教的勧誘、自傷・暴力
- 医療・投資助言、商品広告
- 読みが曖昧な漢字、固有名詞に依存する文章

## Tier(モーラ帯域は DECISIONS.md D-003)

| Tier | モーラ | 出現比率(既定) |
|------|--------|------|
| short | 4〜7 | 20% |
| standard | 8〜12 | 65% |
| long | 13〜26 | 15% |

モーラ数は書かない。`countMora()` が読み仮名から自動計算する。

## 追加手順

1. `packages/phrase-content/src/phrases.ts` の `PHRASE_SEEDS` に追記

```ts
{ id: "daily_0009", displayText: "窓の外を眺める", readingKana: "まどのそとをながめる", tier: "standard", category: "日常" },
```

2. `npm test` を実行。以下が自動検証される:
   - ID・表示文の重複
   - 空文字、英字・記号の混入
   - 読み仮名が全て入力可能か(受理グラフが構築できるか)
   - Tier とモーラ数の一致
   - 表示文20文字以内

## 読み仮名の注意

- 助詞「は」→「は」ではなく発音どおり「わ」…ではなく、**タイピング入力は表記どおり**。「こんにちは」は「こんにちは(konnnichiha)」
- 「を」は wo
- カタカナ語はそのまま書いてよい(自動でひらがなに正規化)。長音は「ー」
- 数字は使わない(読みが割れるため)

## 妨害文章

`GARBAGE_PHRASE_SEEDS` に追加。落ち着かせる・集中させる内容の前向きな文章とする(仕様書 §12.1)。
